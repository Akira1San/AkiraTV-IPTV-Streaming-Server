🎯 Overview

The Fast Scheduler eliminates the complexity of creating and managing JSON schedule files by generating dynamic TV schedules directly from your video collections. It provides intelligent crash recovery, automatic content insertion, and seamless integration with existing AkiraTV workers.

Key Innovation: No more manual JSON editing! The Fast Scheduler creates schedules automatically from your existing video collections, making channel setup effortless and dynamic.

Integration Flow

  1. Schedule Creation: User creates Fast Schedule via web interface with selected collections and settings
  2. Entry Generation: Fast Scheduler loads videos and generates timestamped schedule entries
  3. Checkpoint Save: Schedule automatically saved to checkpoint file for persistence
  4. Worker Request: Linear worker requests schedule from scheduler.py integration layer
  5. Schedule Delivery: scheduler.py checks Fast Scheduler first, then falls back to JSON files
  6. Playback Start: Worker receives schedule entries and begins video playback
  7. Crash Recovery: On restart, resume position calculated and playback continues seamlessly

📖 Usage Guide

Creating a Fast Schedule

  1. Access Interface: Open AkiraTV web UI and navigate to "Collection & Scheduler Wizard"
  2. Launch Fast Scheduler: Click the "⚡ Fast Schedule" button to open the configuration modal
  3. Select Channel: Choose the target channel from your enabled channels list
  4. Choose Collections: Select which video collections to include in the schedule
  5. Configure Settings: Set start time, duration, bumper frequency, and trailer probability
  6. Generate Schedule: Click "Generate Schedule" to create and save the dynamic schedule
Pro Tip: The Fast Scheduler automatically detects your collection format and loads videos accordingly. It supports both legacy and new AkiraTV collection formats.

Starting a Fast Scheduled Channel

  1. Verify Setup: Ensure channel type is set to "Linear" and Fast Schedule exists
  2. Start Channel: Use normal AkiraTV channel start procedures
  3. Automatic Detection: Worker automatically detects and uses Fast Schedule
  4. Resume Playback: System calculates correct resume position if restarting
Success Indicator: Instead of "No schedule found" errors, you'll see "Using Fast Scheduler for channel" in the logs, indicating successful integration.

🔌 API Reference

Schedule Management

POST /api/fast-schedule/{channel}/generate

Generate a new Fast Schedule for the specified channel

GET /api/fast-schedule/{channel}/info

Get comprehensive schedule information and statistics

GET /api/fast-schedule/{channel}/current

Get current playing entry with resume position

GET /api/fast-schedule/{channel}/upcoming

Get upcoming schedule entries (default: 5 entries)

Checkpoint Management

POST /api/fast-schedule/{channel}/save-checkpoint

Manually save current schedule state to checkpoint

POST /api/fast-schedule/{channel}/load-checkpoint

Load schedule state from existing checkpoint file

Collection Management

POST /api/fast-schedule/{channel}/load-collections

Load videos from specified collection files

GET /api/fast-schedule/collections

Get list of available collection files with video counts

Example Request

// Generate Fast Schedule POST /api/fast-schedule/MyChannel/generate { "collections": ["movies", "tv_shows"], "start_time": "09:00", "schedule_hours": 24, "bumper_frequency": 3, "trailer_probability": 0.3 }

🔧 Troubleshooting

Common Issues

❌ No Videos Loaded

Symptoms: "Loaded 0 videos" message when creating schedule
Causes: Collection files not found, invalid JSON format, or empty collections
Solutions: Verify collection file paths, check JSON syntax, ensure collections contain videos

❌ Schedule Not Found by Worker

Symptoms: "No schedule found for linear channel" in worker logs
Causes: Checkpoint not saved, Fast Scheduler integration not working
Solutions: Regenerate schedule, verify checkpoint file exists, check scheduler.py integration

❌ Incorrect Resume Position

Symptoms: Video starts from beginning instead of correct position
Causes: System time issues, incorrect entry time format
Solutions: Verify system time accuracy, check entry time format (HH:MM)

❌ Bumpers/Trailers Not Playing

Symptoms: Only main content plays, no bumpers or trailers
Causes: No bumper/trailer files configured in Fast Scheduler
Solutions: Add bumper and trailer video files to Fast Scheduler configuration

Debug Information

Check these locations for troubleshooting:

  • Checkpoint Files: user/fast_schedules/fast_schedule_{channel_name}.json
  • Worker Logs: Look for "Using Fast Scheduler" or "No schedule found" messages
  • API Logs: Check Fast Scheduler API endpoint responses
  • Collection Files: Verify user/collections/collections_*.json files

🎯 Performance & Scalability

Performance Characteristics

  • Memory Usage: Schedule entries stored in memory for fast access
  • Generation Speed: Near-instantaneous for most collections (O(n) complexity)
  • Checkpoint Operations: Save/load operations complete in under 1 second
  • Scalability: Supports multiple independent channel schedules
Recommendation: For collections with 1000+ videos, ensure adequate RAM is available. Each schedule entry uses approximately 1KB of memory.
