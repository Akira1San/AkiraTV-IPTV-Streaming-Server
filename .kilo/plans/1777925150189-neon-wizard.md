# Plan: Enhance Episodic Tag System in Collection Wizard

## Background
The current "Episodic" tag is a simple boolean string tag that doesn't convey enough information about a series or season. The request is to enrich the system so each collection marked as episodic can store:
- **Series name** (the show/franchise name)
- **Season number** (0 for specials, 1, 2, 3, ...)

This will allow better organization and explicit control over episodic content in the scheduler.

## Investigation Findings
- The `daypart_scheduler.py` already supports `episodic` content blocks and extracts `season` and `episode` information from video metadata or filenames (SxxExx pattern).
- Collection Wizard stores tags as a list of strings; "Episodic" is one such tag.
- Collections are stored as JSON with fields: id, name, cover, description, genre, tags, year, and videos list.
- Videos within a collection can have a `metadata` dictionary, but currently it's unused in the wizard.
- Adding `series_name` and `season` to the collection level is sufficient for the wizard; scheduler may later be extended to read these.

## Proposed Changes
All changes are confined to `akiratv/collection_wizard.py`.

### 1. Add New UI Fields
- Add two new input fields to the **Collection Details** panel (below "Year:").
  - **Series Name:** Entry field (`ttk.Entry`) – free text.
  - **Season:** Spinbox (`ttk.Spinbox`) – integer from 0 to 99, default 0.
- Placement: After the Year field (row 8 and 9 in the grid).

### 2. Add New Instance Variables
In `__init__`:
```python
self.series_name_var = tk.StringVar()
self.season_var = tk.IntVar(value=0)
```

### 3. Create Widgets in `create_widgets`
After the existing fields loop, insert:
```python
# Series Name
ttk.Label(detail_frame, text="Series Name:").grid(row=8, column=0, sticky="w", padx=5, pady=2)
series_entry = ttk.Entry(detail_frame, textvariable=self.series_name_var, width=50)
series_entry.grid(row=8, column=1, padx=5, pady=2, sticky="ew")

# Season
ttk.Label(detail_frame, text="Season:").grid(row=9, column=0, sticky="w", padx=5, pady=2)
season_spin = ttk.Spinbox(detail_frame, from_=0, to=99, textvariable=self.season_var, width=5)
season_spin.grid(row=9, column=1, padx=5, pady=2, sticky="w")
```
Ensure `detail_frame.columnconfigure(1, weight=1)` already exists.

### 4. Save Fields to Collection
Modify `_save_ui_fields_to_selected_collections()` to store the new data:
```python
# Series name
series_name = self.series_name_var.get().strip()
if series_name:
    collection["series_name"] = series_name
elif "series_name" in collection:
    del collection["series_name"]

# Season (always store as int)
season = self.season_var.get()
collection["season"] = season
```
Place these before the `refresh_collection_list()` call.

### 5. Populate Fields When Selecting a Collection
In `on_collection_select()` (single selection branch), after setting year:
```python
self.series_name_var.set(collection.get("series_name", ""))
self.season_var.set(collection.get("season", 0))
```

### 6. Clear Fields When Deselecting / Clearing
In `unselect_all()`, after resetting other fields:
```python
self.series_name_var.set("")
self.season_var.set(0)
```
Also in `load_collections_file()` after clearing selection, clear new fields.

### 7. Provide Sensible Defaults for New Collections
When creating new collections via `rescan_folder()` or `update_selected_collection()`, set a default season of 0. No need to set series_name.
- In `rescan_folder()` (around line 1764), add `"season": 0` to the collection dict.
- In `update_selected_collection()` (around line 1590), add `"season": 0` to the collection dict.

### 8. Backward Compatibility
All code uses `collection.get("field", default)` so collections without `series_name` or `season` will load correctly. The new fields are optional (series_name may be empty). Season defaults to 0.

## Optional Enhancements (Not in Scope)
- Auto-populate `series_name` and `season` from filename using the existing `_extract_series_and_episode()` helper when scanning a folder.
- Link the Episodic checkbox to enable/disable these fields for better UX.
- Show series/season in the collection list display.

## Questions / Clarifications
- Should the Season Spinbox allow up to 99 or a higher limit? 99 covers most series.
- The Episodic tag remains manually toggled; the new fields do not auto-set it. If you want auto-check when series_name is filled, that can be added later.
