# Plan: Sort Series by Episode Number When Importing

## Goal
When importing videos via folder scan (`rescan_folder`) or adding new videos (`update_selected_collection`), sort the files so that series episodes appear in proper numeric order (e.g., S01E01, S01E02, ...) rather than arbitrary filesystem order.

## Locations to Modify
- `akiratv/collection_wizard.py`

## Changes

### 1. Add episode parsing helper method to `CollectionWizard` class

Add a new method `_extract_series_and_episode(self, filepath)` that returns a tuple `(series_name, season_num, episode_num)`.

**Parsing logic:**
- Extract filename stem (without extension)
- Strip known patterns: year (e.g., `(2020)` or `2020`), quality tags (`1080p`, `720p`, `2160p`, `4K`, `BluRay`, `WEBRip`, `HDTV`, `x264`, `x265`), and extra markers.
- Match patterns in order:
  - `S(\d+)E(\d+)` or `s(\d+)e(\d+)` → season, episode
  - `(\d+)x(\d+)` → season x episode (e.g., `1x01`)
  - `Episode\s*(\d+)` or `Ep\s*(\d+)` → assume season 1
  - `Part\s*(\d+)` or `Part\s*(\d+)\s*of` → assume season 1
  - `(\d+)$` at end of filename → episode number only (assume season 1)
- Fallback: `(0, 0)` if no match
- Return `(series_name_clean.lower(), season_num, episode_num)`

This mirrors the approach in `simple_scheduler.py:2439` (`_extract_episode_number`) but also extracts series name.

### 2. Sort video files in `rescan_folder()` (around line 1733)

After collecting `video_files` list:
```python
video_files.sort(key=lambda p: self._extract_series_and_episode(p))
```

This ensures series are grouped together and episodes in numeric order within each series. Files that don't match any episode pattern will get `(0,0,0)` and appear first (or you can return `("", 0, 0)` to send them to top).

### 3. Sort new videos in `update_selected_collection()` (around line 1548-1552)

After building `new_videos` list (before showing preview), sort:
```python
new_videos.sort(key=lambda p: self._extract_series_and_episode(p))
```

### 4. Optional: Also sort `all_videos_in_folder` before deduplication

To ensure preview order matches final order, sort `new_videos` before the preview (step 3) is sufficient. The preview will show the sorted list.

## Edge Cases & Considerations

- **Non-series files**: Files without episode patterns get `(0, 0, 0)` and appear at the top. Acceptable.
- **Different seasons**: The sort key includes season number, so S02 episodes come after S01 for the same series.
- **Case sensitivity**: Series name lowercased for consistent ordering.
- **Performance**: Parsing each filename is cheap relative to I/O.
- **Backward compatibility**: No changes to saved collection structure; only affects order when collections are created from a scan.

## Benefits

- Users importing series get properly ordered collections without manual reordering.
- Consistent with episode-based scheduling in `simple_scheduler.py`.
