# Plan: Fix AkiraTV Linux Launcher

**Problem**: The script `RUN_AkiraTV.sh` fails with `ImportError: attempted relative import with no known parent package` when running `python3 akiratv/ui/main.py` directly.

**Root Cause**: The `main.py` file uses relative imports (`from ..core_api import CoreAPI`). When Python runs a file directly, it doesn't treat it as part of a package, so relative imports fail.

**Solution**: Modify the launcher script to run the module using the `-m` flag, which properly sets up the package context.

**Changes**:
1. Update `RUN_AkiraTV.sh` to use: `python3 -m akiratv.ui.main` instead of `python3 akiratv/ui/main.py`

**Additional considerations**:
- The script should ensure it's run from the project root directory where `akiratv` package is located
- The script already changes to `SCRIPT_DIR`, which should be the project root containing the `akiratv` folder

**Implementation**:
- Edit `RUN_AkiraTV.sh` to replace the last line with: `python3 -m akiratv.ui.main`
